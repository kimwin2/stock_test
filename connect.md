

전체 파이프라인 구조 (AWS 기준)
우리의 목표는 **"10분마다 주식과 뉴스를 분석해서 최신 JSON 파일을 만들고, 앱(프론트엔드)이 언제든 그 데이터를 가져갈 수 있게 하는 것"**입니다.

1. 공장 작업자: AWS Lambda (람다)
역할: 파이썬 코드를 실행해 주는 클라우드 컴퓨터입니다. 24시간 켜놓을 필요 없이, 우리가 코드를 실행하라고 할 때만 잠깐 켜져서 일하고 꺼지기 때문에 비용이 매우 저렴합니다.

하는 일:

증권사 API와 네이버 뉴스에 접속해 데이터를 긁어옵니다 (크롤링).

가져온 뉴스를 ChatGPT API로 보내서 테마 5개와 요약을 받아옵니다.

테마별로 주식 데이터를 필터링해서 대장주 4개를 뽑습니다.

프론트엔드가 요구한 예쁜 형태의 JSON 데이터로 조립합니다.

2. 알람 시계: Amazon EventBridge (이벤트브릿지)
역할: 주기적으로 알람을 울려주는 타이머입니다.

하는 일: "10분마다 AWS Lambda(작업자)를 깨워라!"라고 명령을 내립니다. 이렇게 하면 우리가 자는 동안에도 10분마다 알아서 파이썬 코드가 실행됩니다.

3. 데이터 창고: Amazon S3 (에스쓰리)
역할: 파일이나 데이터를 안전하게 보관해 주는 인터넷 저장소입니다.

하는 일: AWS Lambda가 방금 완성한 최신 dashboard_data.json 파일을 이곳에 덮어쓰기 형태로 저장합니다.

4. 배달부 (API 창구): Amazon API Gateway (API 게이트웨이)
역할: 프론트엔드(React/Flutter)와 우리 AWS 창고를 이어주는 안전한 출입구입니다.

하는 일: 프론트엔드 앱에서 https://api.my-domain.com/latest-theme 같은 주소로 요청을 보내면, 창고(S3)에 있는 최신 JSON 파일을 꺼내서 앱으로 배달해 줍니다.
